微軟什麼都懂，就是沒發現大家已經不要了

納德拉接手前的那幾年，微軟大概是整個科技業最「確定」的一家公司。它太清楚這個世界是踩在 Windows 上跑的——清楚到 iPhone 都上市、Android 都鋪滿了，它還照自己的節奏，慢條斯理把 Windows Phone 做出來。東西不算差，工程實力也還在，問題出在更前面一格：它太確定自己早就知道大家要什麼，於是沒人真的回頭去確認一件事——外面那群人，早就改用了別的東西、過上另一種日子了。

這類巨人翻身的回憶錄我讀過不少，多半把功勞含含糊糊算給「文化」兩個字就帶過；《刷新未來》難得的地方，是它讓你看清那個「確定」，怎麼一格一格把一家好公司推到牆角。納德拉開的解藥只有一句：別再當一個什麼都懂的人，去當一個什麼都想學的人——他把這叫做從 know-it-all 換成 learn-it-all。聽起來像企業文化的場面話，可你把它翻成創業的語言，它立刻變得很不客氣：它戳的，正是一個創業者身上最貴的那個毛病。

你那個點子，就是你的 Windows

每個創業者心裡，都有一個自己的 Windows——那個你最有把握、最不肯被質疑的點子。你對它的篤定，跟當年微軟對 Windows 的篤定，是同一種溫度：大家一定需要這個、這痛這麼明顯怎麼可能沒人要、等我做出來就知道了。愈是確定的東西，你愈不會回頭檢查，因為檢查它會讓你渾身不對勁。於是它就成了你整張創業藍圖裡，唯一沒被驗過、卻撐著全部重量的那一格。

微軟有十萬人的家底接得住那一次失準，賠掉一支手機，還能換個執行長、重新整理一次。你沒有。你那個沒人要的 Windows 一旦垮了，賠掉的可能就是你辭職那年的積蓄，和那兩年回不來的時間。

learn-it-all 翻成創業的話，就是走出門去問

納德拉要微軟學的那個姿勢，拆開來看一點都不玄。一個 know-it-all 碰上市場，習慣是「我來告訴你，你需要什麼」；一個 learn-it-all 碰上市場，習慣是「我來搞清楚，你已經在為什麼掏錢」。前者把力氣花在證明自己對，後者把力氣花在拿自己的假設去撞現實。

痛點這把尺，說穿了就是 learn-it-all 的姿勢長出來的一套動作：別問人家「你會不會想要」——那只是在邀請對方對你說客套話；去看他「現在已經為這件事花了多少錢、忍了多少麻煩」。嘴上說想要，誰都會；真金白銀花下去、特地繞路去解決的，才是市場替你按下的那個「確定」。你想替點子求一個「有人要」的證據，得到外面這群人身上找，不在你自己腦袋裡那場一個人的會議。

把「我確定有人要」，改寫成一個會被打臉的數字

微軟那份「確定」最可怕的地方，是它好多年都沒辦法被證明是錯的——財報還在賺、Windows 還裝在幾億台電腦上，數字一片大好，剛好讓那個過時的確定，又多撐了好幾年。你的「確定」也一樣滑溜：「大家都需要」「市場很大」「總會有人要」，這些話怎樣講都對，永遠不會有哪一天逼你承認自己看走眼。

learn-it-all 的下一步，就是把這種賴不掉的樂觀，壓成一個賴得掉就算輸的數字：到今天為止，有幾個你叫得出名字的真人，真的為這個點子掏過錢？把日期釘死，三個月後再翻開來對一次。

微軟賠掉一整支手機，才肯把那行字劃掉

講句公道話，《刷新未來》本來就不是一本教你怎麼驗證需求的工具書。它是一個巨人回頭認錯、重新整理自己的故事；幫你確認有沒有人要，從來不在它的守備範圍。它真正能塞給你的，是那個姿勢本身：承認「我可能根本不知道大家要什麼」，然後真的放下身段，走出去學。而微軟，是付了好幾年的學費、賠掉一整條產品線，才肯把「我們最懂」那行字劃掉、重抓一份回來的。你手上那行字還沒燒到那麼貴，這正是你最便宜的一次機會。

今晚，先把你最不肯被碰的那句話寫下來

今晚做一件很小的事：把你對這個點子最有把握、最不准別人碰的那句話，原原本本寫下來——通常就是「大家一定需要這個」這一類。寫完，在它旁邊標一行字：這是我的 Windows。然後做那個只有 learn-it-all 做得出來的動作：這個禮拜，別再多加一個功能去證明自己是對的，去找一個活生生、叫得出名字的人，把那句話當成一個真問題丟給他，看他怎麼接。他要是早就在為這件事掏錢、繞路、將就，你的確定就踩到了第一塊真的地基；他要是禮貌地點點頭、然後話題就滑走了，那你也只花一個下午，就買到了微軟拿好幾年才換到的那一課。
