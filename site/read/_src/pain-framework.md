# P.A.I.N. 框架：檢查一個點子值不值得做的四個問題

市面上太多書、太多課，劈頭就叫你「去找痛點」。但很少有人告訴你：找到之後，怎麼判斷這個痛，值不值得你押上時間和錢？

這篇，就是要把那把尺交給你。

我把它叫做 **P.A.I.N.**——四個字母的英文字首拼起來，正好是一個字：**痛**。這不是巧合。你要找的那門值得做的生意，起點永遠是一個真實的痛；而檢查它值不值得做的那把尺，也叫痛。

先說清楚一件事：**這四個問題，不保證你成功。** 世上沒有那種公式，誰跟你說有，你要小心。它能幫你的，是在你動手、花大錢之前，先看清楚——這個點子站不站得住，以及，你下一步該先去問清楚什麼。

## P — People：誰在痛？

具體是哪一群人？不是「大家」，更不能只有「你自己」。

這是最多人跌倒的第一步，包括我。因為自己痛，就理所當然地以為全世界都跟自己一樣痛。但「我痛」這件事，只能證明這世界上至少有一個人在乎——那個人是你。還有沒有第二個，是完全不同的問題。

> 先問自己：這群人，我描述得出輪廓嗎？我現在認識、或找得到幾個真實的人，可以直接去問？

## A — Agony：痛多深？

他為這個問題，已經付出了什麼代價？痛到願意改變、願意付錢嗎？

痛點最實在的樣子，不是一種情緒，是一筆**算得出來的代價**——時間、錢、心情。「我覺得這個好麻煩」是一句抱怨；「我每年浪費幾十個晚上、上萬塊在這件事上」才是一個痛點。前者你會聳聳肩，後者你會想做點什麼。

> 最強的訊號，是他**已經在花錢**找別的方法解。不要問他「你會不會買」，去看他已經為這個問題，付出了什麼。

## I — Intervention：你怎麼介入？

你的解法，有沒有真的把那個最關鍵的痛，挪輕一點？

重點不是功能多，是痛變輕。很多人一頭栽進「我要怎麼做」，把功能堆得又多又滿，卻沒回答最簡單的問題：你讓哪一個「本來會讓人放棄」的卡點，幾乎消失了？

> 還要再問一句：為什麼是你？你手上已經有什麼技能、作品或通路，讓你做得成這件事？

## N — Numbers：數字成立嗎？

最白話的版本：進來的錢，能不能 cover 出去的錢，和你的時間？

數字要一開始就想，不是最後才算。一個客戶、一筆訂單，你大概能收多少？你的成本加上時間是多少？這兩個數字擺在一起，算不算得過來？很多看起來很美的點子，死在這一格——因為從頭到尾，變現從沒被認真設計過。

> 算過、而且問過真的有人願意付，比心裡有個模糊的好感覺，踏實一百倍。

## 順序很重要：P → A → I → N

這套框架真正的關鍵，不只在那四個問題，更在**順序**。

我自己的故事，就是一齣把順序整個做反的悲劇。我是從 **I** 開始的——一頭栽進「我要怎麼做」，把產品、功能、行銷一個一個做到極致；然後偶爾回頭瞄一眼 **N**（數字），發現錢一直在燒；而最該排在最前面的 **P**（誰在痛）和 **A**（痛多深），我幾乎是到了第四年、賠掉近五百萬之後，才第一次認真去問。

把順序顛倒過來——先愛上自己的解法（I），再回頭找誰會要（P）——就是我，也是九成創業者，賠掉時間和錢的那條路。

正確的順序是：**先確認有人痛、痛得夠深（P、A），再決定你怎麼介入、划不划算（I、N）。** 先有人痛，才談你怎麼做；先確認有人要，才花錢把它做出來。

## 怎麼用這把尺

很簡單。拿一個你現在最想做的點子，照 P → A → I → N 的順序，一題一題，誠實地填。

規則只有一條：**哪一題你填不出來、或只能寫「我猜……」，就先別花大錢——那一題，正是你動手前該先去問清楚的事。**
