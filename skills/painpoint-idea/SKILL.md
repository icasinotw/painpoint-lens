# painpoint-idea — 把 HBR 等經典好文「講成人話」放進 /idea

## 這支 skill 在做什麼(以及跟 /lens 拆書差在哪)

山姆是《哈佛商業評論》的重度讀者。HBR 那些經典文章(波特〈策略是什麼?〉、克里斯汀生的破壞式創新、康納曼的決策偏誤……)本身價值就很高、但艱深難啃,一般讀者不容易讀懂。`/idea` 「經典觀念」區的任務,就是**把這些艱深好文的核心觀念,改寫成一般人也讀得懂的白話版本**,放上 painpoint.tw,幫讀者進門。

**這跟 `/lens` 拆書本質不同,所以是獨立一支 skill:**
- `/lens` 拆書 = 拿《痛點》的 P.A.I.N. 之尺,一格一格去**戳一本商業書**(有判斷、挑它的空格)。
- `/idea` 經典觀念 = 當一位**把經典講清楚的好老師**,幫讀者讀懂一位大師的觀念。**姿態是「提供價值」,不是批評。**

> ⛔ **鐵則:`/idea` 預設不套《痛點》P.A.I.N. 世界觀去批評每篇文章。** HBR 這些文章內容價值本來就很高,不需要被戳。你可以在文末輕輕加一句「這觀念在台灣中小企業怎麼用」的延伸,但那是**加值**,不是唱衰。想戳、想挑空格的衝動,留給 `/lens`。

## 🔒 最高鐵則:版權紀律(這區存在的前提,任何一條沒守就不准輸出)

HBR 的文章是有著作權的。這區能做、能長久做的唯一前提,是**分清「觀念」和「表達」**:

1. **只教觀念,絕不重製表達。** 觀念、事實、方法論(例:波特說「策略是取捨」、克里斯汀生說「破壞式創新從低階市場切入」)是**思想,不受著作權保護**,可以用**你自己的話、自己的結構、自己的例子**重新講。但**原文的句子、段落順序、案例、金句排列**是「表達」,受保護。→ **絕不翻譯、絕不改寫、絕不「換句話說」那篇文章本身。** 一句話分界:**不是「把那篇文章變好懂」,而是「用那篇文章教我的觀念,自己重寫一篇好懂的」。**
2. **一定具名、一定標出處。** 文中要明確寫出:**這是哪位作者、哪篇文章、發表在哪、大約哪一年。**(例:「哈佛商學院教授麥可・波特(Michael Porter)1996 年在《哈佛商業評論》發表的〈What Is Strategy?〉(策略是什麼?)」)。標出處反而更安全、對 SEO 與 E-E-A-T 也加分——這叫「站在巨人肩膀上做教學與評述」。
3. **要引用原句,加引號、一兩句為限、絕不整段搬。** 合理使用是「少量、加引號、標出處」;整段搬、或搬很多句短句拼起來,都算侵權。**能用自己的話轉述,就別引原句。**
4. **用你自己的例子當教學載具。** 不要照搬原文用的案例當主線(原文若用某公司當例子,你可以「提一句作者拿它當經典例子」是事實、可寫,但**主要的教學例子要用你自己的**——台灣中小企業、在地生活場景最好)。
5. **文末指路原文。** 結尾輕輕請讀者去讀原文(HBR 英文站 hbr.org 或哈佛商業評論中文版),外連一次即可。這讓你的版本是「進門的橋」而非「原文的替代品」——這是版權上最安全、對讀者也最有價值的位置。

> 版權出事的永遠是「重製表達」,不是「教觀念」。寫完自檢一句:把我的版本跟原文並排,骨架、段落順序、案例、金句對不對得上?**對得上就是衍生著作、要重寫;對不上、只有觀念相通,才安全。**

## 語氣憲法:當一位把經典講清楚的好老師

- **價值優先、把讀者當聰明但沒時間的朋友。** 你的工作是「這篇艱深好文,我幫你三分鐘抓住重點」。
- **具體、白話、有畫面。** 抽象觀念一定配一個台灣在地的小例子落地(便當店、手搖飲、巷口診所……)。
- **誠實但不說教。** 不要金句轟炸、不要「其實…」的翻轉腔、不要把每句話都寫成對立句。
- **沿用站上的全形標點與反 AI 味紀律**(見〈收尾〉),但**拿掉 /lens 的「判斷/挑刺」姿態**——這裡是教學,不是評審。

## 輸入 / 自動挑文 + 去重(鐵則)

**兩種進來的方式:**
- **山姆指名某篇**(例:「寫波特〈策略是什麼?〉的 /idea」)→ 直接寫那篇。
- **山姆只說「幫我寫一篇 HBR 文章」**(由我自動挑)→ 打開帳本 [`data/hbr-idea-backlog.md`](../../data/hbr-idea-backlog.md),挑一篇狀態還是 `[ ]`(沒寫過)的來寫。

**🔁 去重鐵則(避免重複寫到同一篇 HBR 文章):動筆前一定兩處都比對——**
1. 帳本 `data/hbr-idea-backlog.md` 裡該篇是不是已標 `[x] 已寫`;
2. `site/idea/_articles.php` 裡有沒有同 slug / 同主題。
**任一命中就換一篇,絕不重複。**

**自動挑文的偏好順序:** 經典、影響力大、一般讀者最有感;**且盡量避開已被 `/lens` 拆書覆蓋過的同主題**(藍海、核心競爭力那類有相近 /lens 書的,優先順位往後,免得跟 /lens 打架);跨類別輪替,別連續都挑策略類。

**寫完務必做三件事(缺一就是去重破口):** ① 帳本對應那行改成 `[x] 已寫 → {slug}` 並 append 到帳本底部「已寫清單」;② 在 `site/idea/_articles.php` 最上面加一筆;③ 建 `site/idea/{slug}.php` + `_content/{slug}.html`。

**讀懂原文再動筆:** 先用 WebSearch / WebFetch 查證**作者、年份、出處與主要論點**(帳本的年份只是備忘,以查證為準),再**用自己的話**重寫。查證是為了正確與具名標對出處,不是為了搬句子。

## 產出(兩個檔 + 在清單加一筆)

`/idea` 跟 `/guide`、`/ask` 一樣走「單一真相來源純陣列」模式(篇數少、本機一篇一篇寫、無雲端平行寫書壓力,**所以不用碎片、不用 seq**):

1. `site/idea/{slug}.php` — 頁面檔。`{slug}` = 英文篇名/觀念 kebab-case(例:What Is Strategy? → `what-is-strategy`)。照 `site/idea/what-is-strategy.php` 改。
2. `site/idea/_content/{slug}.html` — 正文(純語意 HTML)。照 `site/idea/_content/what-is-strategy.html` 改。
3. **在 `site/idea/_articles.php` 這份清單「最上面」加一筆**(最新在前)。欄位:`slug` / `kicker` / `title`(≤32 字)/ `blurb`(≤160 字)/ `category` / `date`。列表頁 `index.php` 與 `sitemap.php` 都讀這份清單自動長,不用手改它們、nav 也不用動(除非要調選單本身)。

### `category`:一律從受控清單挑(跟 /lens、/guide 同一套,別亂開新詞)
**策略 / 創新 / 行銷 / 管理 / 領導 / 行為 / 創業 / 理財 / 生產力 / 敘事 / 其他**
- 一篇只歸一個主分類(卡片右上角識別看它)。清單沒有的主題,對應到最接近的:決策 → 行為;組織 → 管理;談判 → 行銷或領導。

## 檔案範本

**`site/idea/{slug}.php`**(照 `what-is-strategy.php`):設 `$page`(title/desc/path=`/idea/{slug}`/type=article)、`$reading`(kicker=`經典觀念 · {作者}`、heading、published、content 路徑、back=`['/idea','← 全部經典觀念']`、root=`['經典觀念','/idea']`、funnel、prev/next 設 `null`)、`$page['schema'] = [pain_ask_article($cfg,$page,$reading), pain_person($cfg), pain_breadcrumb(...)]`,最後 require head / header / reading / footer。長文外框吃的變數叫 **`$reading`**(名字寫錯正文渲染不出來)。

**`site/idea/_content/{slug}.html`**:`<h1>` + 多個 `<h2 id="...">` + `<p>` + `<strong>` + 需要時 `<blockquote>`。純語意 HTML、**零 Markdown 符號**。

> 📏 **標題硬上限:卡片 `title`／頁面 `heading`／內文 `<h1>` 三處同一句、各 ≤32 字(漢字)、不准出現全形分號「；」。** 兩個節拍用「,」或破折號;第二段論點移回 `blurb`。(`count-title.py` 目前只掃 `/lens`,`/idea` 請人工比對這三處一致且合規。)

## 文章結構配方(教學型)

不套 P.A.I.N. 四格。用最直覺的教學結構:

1. **開場(2～3 段):** 一個讓讀者「對、我也遇過」的日常場景切入(例:開會人人把「策略」掛嘴上),點出這篇經典要解的困惑;**接著具名交代這是哪位作者、哪篇文章、哪一年、哪裡發表**(版權鐵則 2),並允諾「用最白話的方式帶你抓住 N 個重點」。
2. **主體(2～4 個 `<h2>`,一個觀念一格):** 挑原文最核心的 2～4 個觀念,每個一個小標,先白話講清楚那個觀念,**再配一個你自己的台灣在地例子**落地。觀念之間有邏輯遞進最好(重點一鋪重點二)。
3. **(選配)延伸一句:** 這觀念在今天/在台灣中小企業怎麼用、要注意什麼——加值,不批評。
4. **收尾:** 給一個「帶得走的檢查句」(讀者今天就能拿去用的一句自問),**然後指路原文**(外連 hbr.org 或哈佛商業評論中文版一次),並可連 1～2 篇相關 `/lens` 拆書當延伸閱讀。

## 內鏈與導流(比 /lens 更輕)

**CTA 北極星照舊:把讀者留在站上、多讀一篇,不是導購。`/idea` 因為是「純教學、提供價值」,商業 CTA 要比 /lens 更克制:**
- **正文 `/tool` 內鏈預設 0 個**(這些是管理觀念文,不是「快去驗你的點子」的情境,硬接像業配)。
- **正文 `/book`(《痛點》)內鏈預設 0 個**——別把《痛點》硬塞進一篇波特/克里斯汀生的文章,那正好違反「不套痛點世界觀」。
- **鼓勵連 1～2 篇相關 `/lens` 拆書或其他 `/idea`**(這是站內互鏈、留住讀者的正道,例:波特 → 藍海、好策略壞策略)。錨文字只套乾淨名詞片語,絕不把整句做成連結。
- **外連原文一次**(版權鐵則 5):hbr.org 或哈佛商業評論中文版。這一次外連是版權必要 + E-E-A-T 加分,值得。
- 頁尾(訂閱框/相關閱讀/上下篇)由 `partials/reading.php` 自動合成,別自己組;`.php` 的 `funnel` 只寫 title + 一句不提買書的 body,primary 建議指 `/lens` 或 `/idea`(留在站上),`prev`/`next` 設 `null`。

## 交稿前品質閘 + 反 AI 味(每篇必過)

1. **版權自檢(最高閘,見上「最高鐵則」):** 跟原文並排,骨架/段落順序/案例/金句對不對得上?對得上 → 重寫。文中有沒有具名作者+出處?有沒有整段搬?引句有沒有加引號且極少?
2. **第一手價值 beat:** 至少一處是「一位讀了很多這類文章的人才給得出的白話翻譯或在地例子」——AI 摘要抄不來的具體落地。空泛複述原文摘要不算。
3. **反模板自檢:** 把作者名/篇名遮掉,若整篇讀起來像萬用套版,重寫;每篇開場、例子要各長各的臉。
4. **反差句上限 ≤2(跟 /lens 同尺):** 教學文很容易寫成「不是 X,是 Y」的對立腔,那是 AI 指紋。跑 `count-contrast.py` 逐句數,整篇上限 2 句,標題(h1/h2)本身是反差句的也計入;結尾收束句一律不用反差骨架,改用畫面或動作收。

## 收尾(在 Claude Code 一定要實跑)

1. **反差句閘門:** `python3 tools/count-contrast.py site/idea/_content/{slug}.html`(或 `pbpaste | python3 tools/count-contrast.py`),≤2,附「引用→kept/cut→改寫後」清單。
2. **標點全形化:** `python3 tools/fix-punctuation.py site/idea/_content/{slug}.html`;再確認中文行內無殘留半形 `,:;()!?`、無 Markdown 符號(URL 保持半形)。
3. **標題三處人工比對:** 卡片 `title`／`heading`／`<h1>` 同一句、各 ≤32 字、無「；」。
4. **`php -l`** 對新/改的 `.php`(或全 site)無誤。
5. **本機渲染:** `php -S localhost:8080 site/router.php` 後 curl `/idea` 與 `/idea/{slug}`,確認 200、標題/結構化資料/內鏈都對。
6. **整則回覆最後一行給部署指令**(獨立 code block、之後不接客套):
   ```
   bash ~/Documents/painpoint-lens/deploy.sh
   ```

## 一句話

HBR 那些艱深的經典好文,我當老師幫讀者讀成人話——只教觀念、一定具名標出處、絕不搬原文、指路原文,把讀者留在站上多讀一篇。
